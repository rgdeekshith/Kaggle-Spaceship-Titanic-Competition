# Spaceship Titanic – Logistic Regression Baseline

This repository contains a complete baseline solution for the [Kaggle Spaceship Titanic competition](https://www.kaggle.com/competitions/spaceship-titanic), using scikit-learn’s `LogisticRegression` with simple preprocessing (imputation, one‑hot encoding, and feature scaling).

The goal of the competition is to predict whether a passenger was transported to an alternate dimension (`Transported` = True/False) based on their personal and travel information.

## Approach

### 1. Data

The project uses the original Kaggle files:

- `train.csv`: contains features + target (`Transported`)
- `test.csv`: contains features only
- `sample_submission.csv`: shows the expected submission format

Place these files under `data/` (or update paths in the code accordingly).

### 2. Features

Two types of features are used:

- **Numerical features**

  - `Age`
  - `RoomService`
  - `FoodCourt`
  - `ShoppingMall`
  - `Spa`
  - `VRDeck`

- **Categorical features**

  - `HomePlanet`
  - `CryoSleep`
  - `Cabin`
  - `Destination`
  - `VIP`

The target is:

- `Transported` (boolean)

### 3. Preprocessing pipeline (implemented manually, no sklearn Pipeline)

The preprocessing is done in explicit steps:

1. **Split features and target**

   - Drop `Transported` from `train` to create `X_full_uncleaned`
   - Keep `y = train['Transported']`

2. **Impute missing values**

   - Numeric: `SimpleImputer(strategy="median")`
   - Categorical: `SimpleImputer(strategy="most_frequent")`

3. **Combine and encode categorical variables**

   - Concatenate imputed numeric and categorical DataFrames
   - Apply `pd.get_dummies(..., columns=categorical_features, drop_first=True)` for one‑hot encoding

4. **Scale numerical features**

   - Use `StandardScaler` fitted on the training numerical columns
   - Replace only the numerical feature columns with their scaled versions

This exact sequence is applied to `train`, and the same fitted imputers and scaler are reused for `test`.

---

## Model

- **Algorithm**: `LogisticRegression` from scikit-learn
- **Key parameters**:
  - `max_iter=1000`
  - `random_state=10`

The model is trained on the fully preprocessed `train.csv` (all rows), then used to generate predictions for `test.csv`.

---

## Prediction and submission

Steps for generating predictions:

1. Load and preprocess `test.csv` using:
   - The same numeric and categorical imputers (`imp_num`, `imp_cat`)
   - The same one‑hot encoding logic (`pd.get_dummies` with the same columns list)
   - Column alignment with training features using `DataFrame.align` (to guarantee same columns and order)
   - The same scaler (`StandardScaler`) fitted on the training data

2. Generate predictions:

   - `y_pred = logreg.predict(X_test_scaled)`

3. Build the submission DataFrame:

   ```python
   submission = pd.DataFrame({
       "PassengerId": passenger_ids,
       "Transported": y_pred
   })
   submission.to_csv("submission.csv", index=False)
   ```

4. Upload `submission.csv` to the Kaggle competition page under **Submit Predictions**.

## How to run

### Option 1: Notebook

1. Open `notebooks/spaceship_titanic_logreg.ipynb` in Jupyter or Kaggle.
2. Make sure `train.csv` and `test.csv` are available in the `data/` directory (or adjust paths).
3. Run all cells.
4. A `submission.csv` file will be created in the project (or working) directory.

### Option 2: Script (optional)

If you have `src/train_and_submit.py` mirroring the notebook logic:

```bash
cd spaceship-titanic-ml
python src/train_and_submit.py
```

This should generate `submission/submission.csv`.
